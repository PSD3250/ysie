# AI 코멘트 규칙 (AI COMMENT RULES)

> ⚠️ **AI 필독**: 이 파일은 `script.js`의 `generateSectionComments()` 및 `generateOverallComment()` 함수 내 프롬프트 규칙을 문서화한 것입니다.
> 프롬프트 수정 시 이 파일도 반드시 동기화하세요.

---

## 1. 공통 규칙 — 학년별 톤앤매너 (`getGradeTone`)

| 학년 | 톤앤매너 |
|------|---------| 
| 초등 (초1~6) | 따뜻하고 친근한 말투. 어려운 용어 금지. 칭찬 먼저·충분히. 개선점은 "다음엔 이렇게 해보면 어떨까요?" 식으로 제안. 항상 격려로 마무리. 성취레벨 표현은 "부진", "미흡" 같은 직접적 단어 대신 "조금 더 노력이 필요합니다"처럼 부드럽게. |
| 중등 (중1~3, 기본값) | 직접적이되 존중하는 톤. 부족한 부분 명확히 지적. 도전 의욕을 불러일으키는 언어. 학생 스스로 목표를 세울 수 있도록 구체적 방향 제시. 성취레벨 표현은 동기를 꺾지 않으면서 현실적 수준 전달. |
| 고등 (고1~3) | 전문적이고 간결한 어조. 수능/내신 감안한 실질적 학습 전략. 격려는 한 문장으로 간결하게. 분석·학습 방향에 집중. 성취레벨 표현은 백분위·수준에 맞는 직접적·객관적 표현. |

### 모든 학년 공통 말투 규칙
- **격식 존댓말 필수**: `~ㅂ니다 / ~습니다` 형식만 허용
- **해요체 절대 금지**: `~요, ~네요, ~거예요` 등 금지
- **반말 절대 금지**
- **문단 사이 빈 줄 금지**

## 3. 성취레벨 계산 기준

### 3-1. 백분위 계산 방식
- 해당 영역/총점 기준으로 전체 시험 응시 학생 중 **상위 백분위** 산출
- 공식: `(나보다 높은 점수 학생 수 / 전체 학생 수) × 100 + 1` (최소 1%, 최대 100%)

### 3-2. 성취레벨 (7단계)
| 전체 상위 백분위 | 성취레벨 |
|----------------|---------| 
| 1~10% | 매우 우수 |
| 11~20% | 우수 |
| 21~35% | 다소 우수 |
| 36~55% | 보통 |
| 56~70% | 다소 부진 |
| 71~85% | 부진 |
| 86~100% | 매우 부진 |

> 전체 평균 대비 점수 차이(±)도 함께 AI에게 전달하여 맥락 있는 판단 가능하도록 함.

### 3-3. AI에게 전달되는 성적 데이터 예시 (영역별)
```
개인 점수: 83점 / 영역 만점: 100점 / 전체 평균: 65.2점(전체 대비 +17.8점) /
성취레벨: 우수 / 전체 상위 백분위: 약 14% /
권장학급(중급반) 평균: 72.4점(학급 평균 대비 +10.6점) / 권장학급 내 상위 백분위: 약 31%
```

### 3-4. AI에게 전달되는 총점 데이터 예시 (종합)
```
개인 총점: 265점 / 시험지 만점: 300점 / 전체 평균: 210.5점(전체 대비 +54.5점) /
정답률: 88.3% / 성취레벨: 우수 / 전체 상위 백분위: 약 14% /
권장학급(중급반) 총점 평균: 240.1점(학급 평균 대비 +24.9점) / 권장학급 내 상위 백분위: 약 28%
```

> **🔑 핵심**: 학급 평균 대비 ±점은 JS가 직접 계산해서 프롬프트에 명시합니다. AI가 스스로 높다/낮다를 판단하게 두지 않습니다.

## 4. 영역별 코멘트 규칙 (`generateSectionComments`)

### 작성 구조
1. **잘한 점** (2문장) — 전체 상위 백분위 + 권장학급 내 상위 백분위를 **활용하여** 구체적으로 서술
2. **미흡한 점 / 보완 포인트 / 현재 수준 유지** (1문장) — **JS가 직접 판단해서** 아래 3단계 중 하나 적용

| 조건 | 2번 항목 지시 |
|------|-------------|
| 만점 (개인 = 만점) | "현재 수준 유지" — 미흡한 점·부족한 점 절대 금지. 백분위 활용해 유지 중요성 서술 |
| 학급 평균 이상 (개인 > 학급 평균) | "보완 포인트" — "미흡", "부족", "발전 여지" 같은 부정 표현 절대 금지. 백분위 활용해 만점 대비 부족분만 서술. 세부 데이터 있으면 취약 세부 영역 추가 명시 |
| 학급 평균 미만 (개인 ≤ 학급 평균) | "미흡한 점 또는 약점" — 백분위 활용해 서술. 세부 데이터 있으면 취약 세부 영역 명시. 세부 데이터 없으면 세부 유형·문법 항목 절대 추측 금지 |

3. **구체적 학습 방향 제시** (1문장)
   - 세부 데이터 **있는 경우**: 취약 세부 영역 중심 제시
   - 세부 데이터 **없는 경우**: 해당 영역 전반적 학습 방향만 제시

### ⚠️ 출력 형식 절대 규칙
- 첫 번째 문장은 반드시 **잘한 점에 대한 구체적인 내용**으로 시작
- `"[학생명]의 ~에 대한 피드백입니다"` 같은 **소개·제목 문장 절대 금지**
- **인사말 금지** (안녕하세요 등)
- **과도한 칭찬·축하 표현 절대 금지**: "축하드립니다", "훌륭합니다", "대단합니다" 등
- 영역명을 **영어로 쓰지 말 것** (Grammar, Reading 등 → 문법, 독해 등 한국어만)
- **학생 묘사 시 -시- 존칭 절대 금지**: "받으신", "획득하신", "기록하셨으므로" 등 → "획득하여", "기록했으므로" 형식으로
- **실제 점수와 만점 반드시 언급**
- **전체 상위 백분위 + 권장학급 내 상위 백분위 반드시 코멘트에 활용**
- 호칭은 `"[학생명] 학생은"` 형식만 허용
- **학원명, 교재명, 브랜드명 절대 금지**
- 모든 답변은 **순수 한국어**로 작성
- 🚨 **세부 영역 데이터가 없으면 세부 유형·문항 유형·문법 항목을 절대 지어내거나 추측하지 말 것**

## 5. 종합 코멘트 규칙 (`generateOverallComment`)

### 작성 구조
1. 각 영역 코멘트에서 이미 언급된 세부 내용은 **그대로 반복하지 말 것**
2. 영역들을 가로질러 보이는 **전체적 패턴이나 공통 특징만** 종합적으로 언급 (1~2문장)
3. 부족한 영역의 핵심 학습 방향을 **종합 관점**에서 간결하게 제안 (1~2문장)
4. **전체적 격려 메시지**로 마무리 (1문장)

### ⚠️ 출력 형식 절대 규칙
- 첫 번째 문장은 반드시 **전체 성적에 대한 종합적 내용**으로 시작
- **소개·제목 문장 절대 금지**
- **인사말 금지** / **과도한 칭찬·축하 표현 절대 금지**
- 영역명을 **영어로 쓰지 말 것**
- **학생 묘사 시 -시- 존칭 절대 금지**
- **실제 총점과 만점 반드시 언급**
- **학급 평균 대비 ±점 명시** (JS가 직접 계산해서 프롬프트에 포함)
- 호칭은 `"[학생명] 학생은"` 형식만 허용
- **학원명, 교재명, 브랜드명 절대 금지**
